Version 2.8.2

What's New in Version 2.8.2

Fixed SDK Advanced Sample 에서 NcgHttpRequestDelegate 프로토콜에 대한 델리게이트가 로컬웹서버에 설정이 안되는 오류가 수정됩니다.

  • 델리게이트(setHttpRequestDelegate:) 설정은 반드시 initialize:함수를 호출한 이후여야 합니다.
  • 그렇지 않으면 로컬웹서버에 델리게이트 설정이 안되어 프로토콜이 호출되지 않습니다.

Version 2.7.0

What's New in Version 2.7.0

Changed PallyConTokenInfo 인터페이스 이름이 Ncg2TokenInfo로 변경됩니다.

  • NCG iOS SDK 와 PallyCon FPS SDK를 한 프로젝트에서 같이 사용할 때, PallyConTokenInfo 이름이 동일하여 컴파일 오류가 발생합니다.
  • 이를 피하기 위해 NCG iOS SDK는 인터페이스 이름을 Ncg2TokenInfo로 변경합니다.

Changed iOSNcgSDK.framework에서 시뮬레이터용 라이브리가 제거됩니다.

  • framework 에 시뮬레이터용 라이브러리가 포함된 경우 빌드 에러가 발생하여 제거합니다(Xcode 12.4).

Version 1.1.0.20161018 (released 2016-10-18)

What's New in Version 1.1.0

Changed Enable Bitcode

  • Bitcode를 지원하도록 배포했으나, 써드파티라이브러리에서 Bitcode 지원에 오류가 있어 Bitcode 지원이 안됩니다.
  • App이 Bitcode 지원을 원하도록 되어있다면 Xcode Build Options에서 Enable Bitcode를 No 로 변경하시기 바랍니다.

Changed HTTP 통신 curl 라이브러리로 교체

  • IPv6 대응에 따라 HTTP 통신을 curl 라이브러리로 변경하였습니다.
  • crul 라이브러리 버전은 v7.50.1 입니다.

Added curl 라이브러리 사용에 따른 libz.tbd 라이브러리 프로젝트 추가 필요

  • HTTP 통신을 위한 curl 라이브러리는 libz.tbd 라이브러리를 필요로 합니다.
  • Link Binary With Libraries에서 libz.tbd추가하여 빌드하시기 바랍니다.

Info iOS8 이상 시뮬레이터에서 SDK 동작 안됨

  • 내부 라이브러리가 iOS8 이상을 지원하지 않아 부득히하게 iOS8 이상에서는 시뮬레이터에서 테스트가 불가합니다.
  • 추후 내부 라이브러라가 업데이트되면 변경하여 배포됩니다.
  • 시뮬레이터가 아닌 디바이스에서는 정상 동작하므로 실제 디바이스에서 테스트하시기 바랍니다.

Deprecated -(void) setMultipleConnectionDeny:(BOOL)enabled 제거

  • iOS 기본 플레이어는 로컬 서버에 데이터를 요청할 때 멀티로 요청하므로 Multiple Connection을 제한하면 재생이 안되는 문제가 발생하여 이 API를 제거합니다.

Version 1.0.1.20160118 (released 2016-01-18)

What's New in Version 1.0.1.20160114

Fixed [Ncg2Agent getNcg2HeaderInfo] 버그 수정

  • getNcg2HeaderInfo: 함수에서 open한 파일에 대해 close가 없어 여러번 함수 호출 시 런타임 에러가 발생합니다. open한 파일을 사용 후 close 하도록 수정하였습니다.
  • 직접 호출하지 않더라도 파일 검사, 라이선스 확인 및 획득 등에서 호출되어 앱 동작에 문제가 되므로 배포된 버전을 적용해야 합니다.

Fixed External Display(AirPlay) 버그 수정

  • 이전버전에서 AirPlay 경우 라이선스의 External Display 설정 다르게 동작하는 버그가 있었습니다.
  • 라이선스에 External Display 설정과 맞게 동작하도록 버그를 수정했습니다.
