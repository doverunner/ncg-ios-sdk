Version 2.9.1

What's New in Version 2.9.1

Changed The library header is changed to use Simulator and device without distinction.

  • iOSNcgSDK.xcframework supports Apple Silicon(M1), so Simulator and device library header names are different, but they are integrated into one.

    xxxxxxxxxx
    // Before change
    #import <iOSNcgSDKSimulator/iOSNcgSDK.h>
    #import <iOSNcgSDK/iOSNcgSDK.h>
    

    // after change
    #import <iOSNcgSDK/iOSNcgSDK.h>

 

Version 2.9.0

What's New in Version 2.9.0

Added Added XCFramework to NCG iOS SDK.

  • iOSNcgSDK.xcframework is in the lib folder.

 

Version 2.8.2

What's New in Version 2.8.2

Fixed In the SDK Advanced Sample, an error where the delegate for the NcgHttpRequestDelegate protocol could not be set in the local web server is fixed.

  • The delegate (setHttpRequestDelegate:) must be set after calling the initialize: function.
  • Otherwise, the implemented protocol will not be called because the delegate is not set in the local web server.

 

Version 2.8.1

What's New in Version 2.8.1

Changed The CURL library used inside NCG will be changed.

 

Version 2.8.0

What's New in Version 2.8.0

Added Http Live Streaming content key file protection logic is added.

  • A logic has been added to prevent the leakage of HLS content keys by accessing the local web server.

Added The library for the simulator is added to the iOSNcgSDK.framework.

  • If you have problems building, please select Do Not Embed in the Frameworks, Libraries, and Embedded Content section.

Version 2.7.1

What's New in Version 2.7.1

Added Logs have been added to some functions in NCG SDK, and can be checked by setting Ncg2Agent.setExceptionalEventDelegate: and implementing NcgExceptionalEventDelegate.

Version 2.7.0

What's New in Version 2.7.0

Changed The PallyConTokenInfo interface name is changed to Ncg2TokenInfo.

  • When NCG iOS SDK and PallyCon FPS SDK are used together in a project, a compilation error occurs because the PallyConTokenInfo name is the same.
  • To avoid this, NCG iOS SDK changes the interface name to Ncg2TokenInfo

Changed The simulator library is removed from iOSNcgSDK.framework.

  • If the simulator library is included in framework, a build error occurs(Xcode 12.4).

Version 1.1.0.20161018 (released 2016-10-18)

What's New in Version 1.1.0

Changed Enable [Bitcode][1]

  • Deployed it to support bitcode, but there is an error in the bitcode support in the third party library, So Bitcode is not supported.
  • If your app wants to support BitCode, please change the Enable Bitcode to No in Xcode Build Options.

Changed IPv6 Support([CURL][2])

  • We changed to CURL library for IPv6 support.
  • The CURL library version is v7.50.1.

Added Need to add libz.tbd library project according to CURL library usage

  • The CURL library requires the libz.tbd library.
  • Build Link Binary With Libraries by adding libz.tbd.

Info SDK does not work on iOS8 or higher simulator.

  • Third party library problem, iOS8 or later simulator can not be tested.
  • If the third party library is updated later, it will be changed and distributed.
  • Since it operates normally on non-simulator devices, please test on actual devices.

Deprecated -(void) setMultipleConnectionDeny:(BOOL)enabled

  • The API is removed due to playback issue which is caused by multiple connections from iOS native player.

Version 1.1.0.20160808 (released 2016-08-08)

What's New in Version 1.1.0

Changed uses curl library for HTTP communication.

  • To apply IPv6, HTTP communication is handled by curl library.
  • The version of curl library is v7.50.1.

Changed removes -(void) setMultipleConnectionDeny:(BOOL)enabled

  • The API is removed due to playback issue which is caused by multiple connections from iOS native player.

Version 1.0.2.20160317 (released 2016-03-17)

What's New in Version 1.0.2.20160217

Changed iOS 9.X Support

  • Earlier SDK didn't work with iOS 9.X simulator because of missing library for the simulator.
  • This version contains the library for iOS 9.X simulator.

Changed Enable [Bitcode][1]

  • Bitcode is now supported by the SDK.
  • If you want Bitcode support in your app, please enable bitcode at Xcode Build Options > Enable Bitcode > Yes.

Version 1.0.1.20160118 (released 2016-01-18)

What's New in Version 1.0.1.20160114

Fixed [Ncg2Agent getNcg2HeaderInfo] bug fix

  • A runtime error was caused in getNcg2HeaderInfo: function because of the missing 'close' for opened file. It is fixed now.
  • The function is used by other internal APIs, so please apply this fixed version even if you don't directly use the function.

Fixed External Display(AirPlay) bug fix

  • In earlier versions, external display option in license data didn't work properly for AirPlay.
  • The bug about External Display is now fixed.
